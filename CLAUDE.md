# Портфолио Милы Колгановой

Сайт-портфолио на заказ для Милы Колгановой — режиссёра монтажа / руководителя видеопроизводства (5 лет опыта, ТВ, документальное кино, корпоративные видео). Изначально задача была сформулирована как "портфолио дизайнера", но по факту резюме (hh.ru) показало, что её основная специализация — видеопродакшн, а не дизайн. Сайт переориентирован под её реальный профиль; дизайн (постеры/motion design) упомянут как один из кейсов, а не как основная тема.

## Стек

Чистые HTML/CSS/JS, без сборщиков и фреймворков — статический сайт.

- `index.html` — вся структура и контент одной страницы (хедер, hero, бегущая строка тегов, кейсы, навыки, контакты, футер)
- `css/style.css` — стили, тёмная тема, CSS-переменные в `:root` для цветов
- `js/script.js` — мобильное меню (бургер) + анимация появления блоков кейсов при скролле (IntersectionObserver) + подгрузка видео VK/YouTube/Rutube (плеер во `<iframe>`) или прямого mp4-файла (обычный `<video>`) по клику на превью + подбор цвета рамки превью по самой картинке (canvas)
- `images/` — превью-картинки для видео в кейсах (скачаны как обычные JPG-файлы, а не подгружаются "на лету" — так надёжнее: не ломается при недоступности стороннего CDN)

## Локальный запуск

Через расширение **Live Server** в VS Code: правой кнопкой на `index.html` → "Open with Live Server". Либо `python -m http.server` из этой папки.

## Формат для присылки новых кейсов

Заказчица присылает новые примеры для «Кейсов» в таком виде (пустая строка отделяет название раздела от примеров и один пример от другого):

```
Название раздела

что было сделано
название проекта
ссылка на видео

что было сделано
название проекта
ссылка на видео
```

Когда приходит текст в таком формате, по каждому примеру нужно:

1. Определить источник по ссылке: `vkvideo.ru`/`vk.com` → VK, `youtube.com`/`youtu.be` → YouTube, `rutube.ru` → Rutube, `disk.yandex.ru`/`yadi.sk` → Яндекс.Диск, прямая ссылка на файл `.mp4` (обычно на сайте телеканала/СМИ) → «file», что-то другое (например, ссылка на статью без видео) → нет видео, оформлять как ссылку без плеера (см. пункт 4б).
2. Достать ID из ссылки:
   - VK — `oid` (с минусом) и `id` из `.../video{oid}_{id}`.
   - YouTube — часть после `watch?v=` или после `youtu.be/` (без хвоста вида `?si=...`).
   - Rutube — `id` из `.../video/private/{id}/` (или просто `.../video/{id}/`) и токен доступа `p` из `?p=...`, если он есть (у приватных ссылок).
   - Для прямого `.mp4`-файла ID не нужен — сама ссылка и есть источник видео, но перед использованием стоит проверить в Browser-инструменте (`read_network_requests`), что она отдаёт код 200/206 без подписанного токена с коротким сроком жизни (в отличие от Яндекс.Диска — см. ниже) — иначе через какое-то время ссылка перестанет работать.
3. Скачать превью-картинку в `images/` (не ссылаться на источник напрямую — см. «Стек», пункт про `images/`):
   - YouTube: `https://img.youtube.com/vi/{id}/maxresdefault.jpg` (это настоящий full-bleed кадр 16:9 без чёрных полос; `hqdefault.jpg` — запасной вариант, но у него полосы запечены в саму картинку).
   - VK: прямой ссылки на превью нет — нужно открыть `https://vk.com/video_ext.php?oid={oid}&id={id}&hd=2` (или через Browser-инструмент страницу `vkvideo.ru/video{oid}_{id}`) и найти в HTML JSON с превью, привязанным именно к этому `id` — на странице есть превью и других (соседних/рекомендованных) видео, поэтому искать контекст рядом с `"id":{id},"owner_id":{oid}`, а не первое попавшееся `sun9-...` изображение (на этом уже ловили ошибку).
   - Rutube: открыть `https://rutube.ru/api/video/{id}/?p={p}` (работает и для приватных ссылок с токеном) — в JSON-ответе есть готовое поле `thumbnail_url` с настоящим кадром видео, а также `embed_url` (обычно совпадает с `https://rutube.ru/play/embed/{id}?p={p}`).
   - Яндекс.Диск: `https://cloud-api.yandex.net/v1/disk/public/resources?public_key={ссылка на файл, urlencoded}&preview_size=XXL` — в JSON есть поле `preview` с прямой картинкой-кадром. Прямую ссылку на сам видеофайл (`file`) для встраивания плеера использовать нельзя — она с коротким по времени жизни токеном.
   - Прямой `.mp4`-файл: готового кадра-превью обычно нет, поэтому кадр вытаскивается из самого видео через Browser-инструмент (`javascript_tool`) — создать `<video>`, дождаться `loadedmetadata`, выставить `currentTime` на пару секунд, по событию `seeked` нарисовать кадр в `<canvas>` и получить `canvas.toDataURL('image/jpeg')`; из полученного base64 собрать jpg-файл (например через `python -c "..."`, декодируя base64 в бинарные данные).
   - Ссылка-статья без видео (например, RT DE изначально был только ссылкой на статью) — взять `og:image` со страницы статьи.
   - Имя файла — `images/case-{слаг-проекта}.jpg`.
4. Добавить в `index.html` новый `.case-example`:
   - **а. Если есть видео (VK/YouTube/Rutube/file)** — по образцу уже существующих: картинка-постер, ссылка с `data-video-play` + `data-video-provider="vk"|"youtube"|"rutube"|"file"` и соответствующими атрибутами (`data-vk-oid`/`data-vk-id`, `data-yt-id`, `data-rutube-id`/`data-rutube-p`, или для «file» просто `data-video-src="{прямая ссылка на mp4}"`), текст «что было сделано» (`.case-example__tags`) и название проекта (`.case-example__project`). У «file» по клику вместо `<iframe>` подставляется обычный `<video controls autoplay>` (см. `initVideoFacades` в `script.js`).
   - **б. Если видео нет** (Яндекс.Диск — просто видеофайл без встраиваемого плеера, либо ссылка на статью/страницу) — тот же `.video-facade`, но без `data-video-facade`/`data-video-provider`/`data-video-play`: ссылка просто открывает исходную страницу в новой вкладке (`target="_blank"`) — по клику ничего не подменяется. Для Яндекс.Диска (там реально видео, просто открывается не на сайте) кнопку `.video-facade__play` оставляем; для статьи без видео — `.video-facade__play` не добавляем, чтобы не обещать плеер там, где его нет.
   - Класс `ph--N` можно ставить любой по кругу 1-4 — это только заглушка на время загрузки, реальный цвет рамки JS вычислит сам по картинке.
5. Если название раздела совпадает с уже существующим `.case-section__title` — добавить пример в его сетку. Если раздела с таким названием ещё нет — завести новый `.case-section` (шаблон — комментарий `<!-- сюда добавлять новые .case-section -->` в конце `#cases`). Заголовки разделов — без номеров (не «1. Интервью», а просто «Интервью»), даже если в присланном тексте разделы пронумерованы.
6. Проверить в браузере (превью показывает картинку, по клику играет нужное видео или открывается страница в новой вкладке) и закоммитить.

## Текущий статус и то, что осталось сделать

- Контент (тексты, статистика, навыки) — реальные данные из резюме Милы.
- **Контакты в футере — заглушки** (`mail@example.com`, фиктивный телефон, нерабочий Telegram). Реальные контакты (email, телефон, телеграм) сознательно убраны перед публикацией и хранятся в истории git на случай, если понадобится их вернуть.
- **Раздел «Кейсы» устроен по секциям**: внутри `#cases` может быть несколько `.case-section`, у каждой — заголовок (без номера) и сетка `.case-example`-блоков. Каждый блок — это один пример: превью видео (картинка `.video-facade__poster` из `images/`, вписанная целиком через `object-fit: contain`, поверх — кнопка Play → живой плеер VK/YouTube/Rutube по клику через `data-video-provider="vk"|"youtube"|"rutube"` + соответствующие `data-*`-атрибуты, либо просто ссылка на исходную страницу в новой вкладке, если встроить плеер нельзя, — подробности и когда что использовать см. «Формат для присылки новых кейсов»), под ним — что было сделано (`.case-example__tags`) и название проекта (`.case-example__project`). Цвет рамки вокруг превью (видна там, где `contain` оставляет пустое место сверху/снизу) не фиксированный — JS сам вычисляет его по верхней полосе картинки (`initVideoPosterColors` в `script.js`, через canvas), поэтому у каждого нового превью рамка автоматически получится гармоничной без ручной подгонки цвета. До того, как JS отработает (или если он выключен), виден резервный цвет из класса `.ph--N`.
  - Готовы секции: «Интервью» (Папочка канцлера — VK, Ольга Петерзен и Патрик Бааб — YouTube), «Подкасты» (Папочка канцлера ×2 — VK, Эльдар Лайв — YouTube), «Промо-ролики» (Амбассадоры Сбер, Zobnin team — оба Rutube), «Стримы» (Папочка канцлера — VK), «Прямые эфиры» (RT DE — прямая ссылка на mp4-файл, провайдер «file»), «Документальные фильмы» (Цикл телепрограмм «Навстречу времени» — Rutube; Дипломный документальный фильм — Яндекс.Диск, ссылка без плеера), «Анимированные постеры к кинофильму» (ALTER, Идентификация — оба Яндекс.Диск, ссылка без плеера).
  - Старый плоский грид из 4 карточек-заглушек (Амбассадоры Сбера, Постеры к фильму, KD Studios, а также демо-карточка «Молодая страна» с тестовыми VK-видео) убран по просьбе заказчика — этот контент уже переразложен по секциям выше (Амбассадоры Сбер и постеры к фильму) либо был тестовым и не восстанавливался (KD Studios, «Молодая страна»).
  - Механизм карусели-пролистывания внутри одной карточки (стрелки/точки/свайп) больше не используется — в новом формате у каждого блока ровно одно видео. Код сохранён в истории git (коммиты до текущего), можно вернуть, если понадобится листать несколько медиа в одном блоке.
