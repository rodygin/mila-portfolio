# Портфолио Милы Колгановой

Сайт-портфолио на заказ для Милы Колгановой — режиссёра монтажа / руководителя видеопроизводства (5 лет опыта, ТВ, документальное кино, корпоративные видео). Изначально задача была сформулирована как "портфолио дизайнера", но по факту резюме (hh.ru) показало, что её основная специализация — видеопродакшн, а не дизайн. Сайт переориентирован под её реальный профиль; дизайн (постеры/motion design) упомянут как один из кейсов, а не как основная тема.

## Стек

Чистые HTML/CSS/JS, без сборщиков и фреймворков — статический сайт.

- `index.html` — вся структура и контент одной страницы (хедер, hero, бегущая строка тегов, кейсы, навыки, контакты, футер)
- `css/style.css` — стили, тёмная тема, CSS-переменные в `:root` для цветов
- `js/script.js` — мобильное меню (бургер) + анимация появления блоков кейсов при скролле (IntersectionObserver) + подгрузка видео VK/YouTube по клику на превью + подбор цвета рамки превью по самой картинке (canvas)
- `images/` — превью-картинки для видео в кейсах (скачаны как обычные JPG-файлы из VK/YouTube, а не подгружаются "на лету" — так надёжнее: не ломается при недоступности VK/YouTube CDN)

## Локальный запуск

Через расширение **Live Server** в VS Code: правой кнопкой на `index.html` → "Open with Live Server". Либо `python -m http.server` из этой папки.

## Текущий статус и то, что осталось сделать

- Контент (тексты, статистика, навыки) — реальные данные из резюме Милы.
- **Контакты в футере — заглушки** (`mail@example.com`, фиктивный телефон, нерабочий Telegram). Реальные контакты (email, телефон, телеграм) сознательно убраны перед публикацией и хранятся в истории git на случай, если понадобится их вернуть.
- **Раздел «Кейсы» устроен по секциям**: внутри `#cases` может быть несколько `.case-section` (например, «Интервью»), у каждой — заголовок и сетка `.case-example`-блоков. Каждый блок — это один пример: превью видео (картинка `.video-facade__poster` из `images/`, вписанная целиком через `object-fit: contain`, поверх — кнопка Play → живой плеер VK/YouTube по клику через `data-video-provider="vk"|"youtube"` + `data-vk-oid`/`data-vk-id` или `data-yt-id`), под ним — что было сделано (`.case-example__tags`) и название проекта (`.case-example__project`). Цвет рамки вокруг превью (видна там, где `contain` оставляет пустое место сверху/снизу) не фиксированный — JS сам вычисляет его по верхней полосе картинки (`initVideoPosterColors` в `script.js`, через canvas), поэтому у каждого нового превью рамка автоматически получится гармоничной без ручной подгонки цвета. До того, как JS отработает (или если он выключен), виден резервный цвет из класса `.ph--N`. Чтобы добавить пример в существующую секцию — вставить ещё один `.case-example` в `index.html` (не забыть скачать и положить в `images/` картинку-превью для его видео, класс `ph--N` можно ставить любой — это только заглушка на время загрузки); чтобы добавить новую категорию — новый `.case-section` (шаблон см. в комментарии перед `<!-- сюда добавлять новые .case-section -->`).
  - Готова секция «Интервью» (3 примера: «Папочка канцлера» — VK, «Ольга Петерзен» и «Патрик Бааб» — YouTube).
  - Старый плоский грид из 4 карточек-заглушек (Амбассадоры Сбера, Постеры к фильму, KD Studios, а также демо-карточка «Молодая страна» с тестовыми VK-видео) убран по просьбе заказчика — контент этих кейсов нужно будет заново разложить по секциям, когда до них дойдёт очередь.
  - Механизм карусели-пролистывания внутри одной карточки (стрелки/точки/свайп) больше не используется — в новом формате у каждого блока ровно одно видео. Код сохранён в истории git (коммиты до текущего), можно вернуть, если понадобится листать несколько медиа в одном блоке.
- Из резюме ещё не разложены по секциям: реклама для футбольного клуба, корпоративные ролики для "Норникеля", подкасты/интервью на фрилансе, плюс контент старых убранных карточек (см. выше).
